# 电商购物平台 - 需求分析说明书

## 1. 项目概述

### 1.1 项目背景

本项目为课堂贯穿项目「电商购物平台」的延续与完善，参考成熟电商业务流程，在原有基础上补充并完善多个功能模块，实现用户端购物、订单、支付、评价以及管理员端用户与评价管理，形成完整可用的 B2C 电商系统。

### 1.2 项目目标

- 为用户提供商品浏览、购物车、下单、模拟支付、订单查询与取消、商品评价等完整购物流程。
- 为管理员提供用户管理、评价管理能力，保障平台秩序与数据质量。
- 采用前后端分离架构，界面美观、交互友好、操作便捷。
- 系统采用分层架构，代码规范、可维护可扩展。

### 1.3 术语与缩写

| 术语 | 说明 |
|------|------|
| 用户 | 注册并登录的普通顾客，可购物、下单、评价 |
| 管理员 | 具有 admin 角色的用户，可管理用户与评价 |
| JWT | JSON Web Token，用于前后端无状态认证 |
| 订单状态 | 0 待付款、1 已付款、2 已发货、3 已完成、4 已取消 |

---

## 2. 用户与角色

| 角色 | 描述 | 主要行为 |
|------|------|----------|
| 游客 | 未登录访问者 | 浏览商品、分类、商品详情；需登录后才能加购、下单等 |
| 普通用户 | 已注册且角色为 user | 注册、登录、个人信息、购物车、下单、支付（模拟）、订单查询/取消、商品评价、我的评价 |
| 管理员 | 已注册且角色为 admin | 除普通用户能力外：用户管理（列表、编辑状态/角色）、评价管理（列表、删除） |

---

## 3. 功能需求

### 3.1 用户管理模块

| 功能编号 | 功能名称 | 描述 | 优先级 |
|----------|----------|------|--------|
| U01 | 用户注册 | 用户名、密码、昵称、手机、邮箱；用户名唯一；密码加密存储 | 高 |
| U02 | 用户登录 | 用户名+密码校验；账号禁用则拒绝登录；返回 JWT | 高 |
| U03 | 当前用户信息 | 根据 JWT 返回当前用户基本信息（不含密码） | 高 |
| U04 | 权限控制 | 基于 JWT 的接口鉴权；管理员可访问 /admin/* 等接口 | 高 |
| U05 | 用户管理（管理员） | 用户列表、编辑用户状态（启用/禁用）、编辑用户角色 | 中 |

### 3.2 商品管理模块

| 功能编号 | 功能名称 | 描述 | 优先级 |
|----------|----------|------|--------|
| P01 | 商品分类 | 分类列表（树形或扁平），用于筛选与展示 | 高 |
| P02 | 商品列表 | 分页/列表展示，支持按分类筛选 | 高 |
| P03 | 商品详情 | 单个商品详情（名称、价格、库存、主图、详情等） | 高 |
| P04 | 商品搜索 | 按关键词搜索商品名称 | 中 |

### 3.3 购物车模块

| 功能编号 | 功能名称 | 描述 | 优先级 |
|----------|----------|------|--------|
| C01 | 添加商品 | 将商品加入购物车，可指定数量；校验库存 | 高 |
| C02 | 修改数量 | 修改购物车中某商品数量；不超过库存 | 高 |
| C03 | 勾选/取消勾选 | 勾选项参与结算 | 高 |
| C04 | 删除商品 | 从购物车移除某商品 | 高 |
| C05 | 购物车列表 | 展示当前用户购物车项（含商品信息、价格、小计） | 高 |
| C06 | 结算 | 仅对勾选项生成订单（见订单模块） | 高 |

### 3.4 订单管理模块

| 功能编号 | 功能名称 | 描述 | 优先级 |
|----------|----------|------|--------|
| O01 | 订单生成 | 从购物车勾选项生成订单；填写收货人、电话、地址；扣减库存、清空对应购物车项 | 高 |
| O02 | 订单状态 | 0 待付款、1 已付款、2 已发货、3 已完成、4 已取消 | 高 |
| O03 | 订单查询 | 当前用户订单列表、订单详情、订单明细项 | 高 |
| O04 | 订单取消 | 仅待付款订单可取消 | 中 |

### 3.5 支付模块

| 功能编号 | 功能名称 | 描述 | 优先级 |
|----------|----------|------|--------|
| PAY01 | 模拟支付 | 用户点击「去支付」后，将订单状态更新为已付款，无真实支付渠道 | 高 |
| PAY02 | 支付状态更新 | 订单状态由待付款变为已付款 | 高 |

### 3.6 评价模块

| 功能编号 | 功能名称 | 描述 | 优先级 |
|----------|----------|------|--------|
| R01 | 商品评价 | 用户对已购商品进行评分（1–5 星）与文字评价；可关联订单 | 高 |
| R02 | 评价展示 | 商品详情页展示该商品评价列表 | 高 |
| R03 | 我的评价 | 当前用户查看自己发表的全部评价 | 中 |
| R04 | 评价管理（管理员） | 查看全部评价列表、删除不当评价 | 中 |

## 4. 非功能需求

### 4.1 技术约束

- 后端：Java 语言，Spring Boot + MyBatis，MySQL 持久化。
- 前端：网页版，Vue 3 + Vite + Element Plus，与后端分离。
- 认证：JWT，请求头携带 Token 访问需登录接口。

### 4.2 性能与可用性

- 常规操作（列表、详情、加购、下单）响应时间在可接受范围内。
- 部署方式支持 Docker Compose 一键启动，便于演示与验收。

### 4.3 安全

- 密码不可明文存储，需加密（如 BCrypt）。
- 接口按角色控制访问；未登录或无权限返回 401/403。

### 4.4 数据

- 数据库使用 MySQL，字符集 utf8mb4，支持中文及常见符号。
- 关键业务（下单、支付、取消）具备事务一致性。

## 5. 业务流程简述

### 5.1 用户购物流程

1. 注册/登录 → 2. 浏览分类与商品 → 3. 加入购物车（可修改数量、勾选）→ 4. 结算并填写收货信息 → 5. 生成订单 → 6. 模拟支付 → 7. 可选：对商品进行评价。

### 5.2 订单状态流转

- 待付款(0) → 已付款(1)（模拟支付）→ 已发货(2) → 已完成(3)；待付款可取消(4)。

### 5.3 管理员操作

- 用户管理：查看用户列表，禁用/启用账号，修改用户角色。
- 评价管理：查看全部评价，删除违规评价。

---

## 6. 需求优先级汇总

- **高**：注册、登录、当前用户、权限、分类/列表/详情、购物车全流程、下单、支付（模拟）、订单查询与状态、商品评价与展示、我的评价、管理员用户与评价管理。
- **中**：商品搜索、订单取消、评价管理（列表与删除）。

以上需求均在当前系统中实现并可用。
