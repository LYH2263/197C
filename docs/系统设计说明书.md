# 电商购物平台 - 系统设计说明书

## 1. 系统架构概述

本系统采用**前后端分离**的 B/S 架构：前端为 Vue 3 + Vite + Element Plus 单页应用，后端为 Spring Boot + MyBatis 的 RESTful API 服务，数据持久化采用 MySQL。用户通过浏览器访问前端，前端通过 HTTP 请求调用后端 API，后端通过 JWT 进行无状态认证与鉴权。

### 1.1 技术栈

| 层次 | 技术选型 | 说明 |
|------|----------|------|
| 前端 | Vue 3、Vite、Element Plus、Vue Router、Pinia、Axios | 单页应用，组件化开发 |
| 后端 | Java 17、Spring Boot 3、Spring Security、MyBatis | REST API、JWT 认证 |
| 数据库 | MySQL 8.0 | utf8mb4，InnoDB |
| 部署 | Docker、Docker Compose、Nginx | 一键构建与运行 |

### 1.2 分层架构

系统严格采用分层结构，职责分明、耦合度低：

| 层次 | 包路径 | 职责 |
|------|--------|------|
| 表示层（UI） | `controller`、前端 `views` | 接收请求、参数校验、调用 Service、返回统一 Result |
| 业务逻辑层 | `service` | 业务规则、事务控制、调用 Mapper |
| 数据访问层 | `mapper` + XML | 与数据库交互，CRUD |
| 实体层 | `entity`、`dto` | 领域实体、请求/响应 DTO |
| 通用层 | `common`、`config`、`security` | 统一响应、异常处理、安全配置、JWT |

## 5. 模块设计简述

### 5.1 用户与认证

- **AuthController**：`/auth/register`、`/auth/login`、`/auth/me`。
- **AuthService**：注册（用户名唯一、密码加密）、登录（校验密码与状态）、获取当前用户信息。
- **JwtService**：生成与解析 JWT；JwtAuthenticationFilter 在请求链中解析 Token 并设置 SecurityContext。

### 5.2 商品与分类

- **ProductController**：商品列表（可分页、按分类、关键词）、商品详情；分类列表。
- **ProductService / CategoryService**：封装 Mapper 查询，无复杂业务。

### 5.3 购物车

- **CartController**：加购、列表、修改数量、勾选、删除。
- **CartService**：加购时校验商品与库存；列表聚合商品信息与小计。

### 5.4 订单与支付

- **OrderController**：创建订单、订单列表、订单详情、订单明细、模拟支付、取消订单。
- **OrderService**：创建订单时校验勾选与库存、生成订单号、写 order_main/order_item、扣库存、删购物车项；支付与取消仅更新订单状态。

### 5.5 评价

- **ReviewController**：提交评价、商品评价列表、我的评价；管理员评价列表与删除。
- **ReviewService**：校验用户与订单/商品关系，写 review 表；管理员删除不校验订单归属。

### 5.6 管理员

- **AdminController**：用户列表、更新用户状态/角色；评价列表、删除评价。
- **AdminService**：调用 UserMapper、ReviewMapper 完成上述操作；权限由 Spring Security 按角色限制路径。

### 5.7 异常与安全

- **GlobalExceptionHandler**：统一处理 IllegalArgumentException、IllegalStateException、MethodArgumentNotValidException、BindException 及通用 Exception，返回统一 Result。
- **SecurityConfig**：放行 `/auth/login`、`/auth/register`、商品与分类的公开接口；其余需认证；`/admin/**` 需 ADMIN 角色。

---

## 6. 设计原则体现

- **单一职责**：Controller 只负责入参出参，Service 只负责业务，Mapper 只负责数据访问。
- **开闭原则**：通过分层与接口（Mapper）扩展数据源或实现，无需修改上层。
- **依赖倒置**：Controller 依赖 Service 抽象（当前为具体类），Service 依赖 Mapper 接口，便于测试时 Mock。
